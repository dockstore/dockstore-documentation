Reasoning: We want to break up our index page with some images so it isn't just a wall of text. If we want to avoid using full-size logos (which have different sizes and aren't very neat when placed side by side, and some of them do not have high-res images easily available), we can use square logos. With the goal of square logos in mind, we originally wanted to use figures (images with captions) in this section, so that logos besides Terra and Galaxy's can immediately be understood by saying what they actually represent as a caption. But, Sphinx does not support putting figures (images with captions) side-by-side.

So instead, we cheat a little bit by rendering the final result in HTML, where we do not have to work with the limitations of Sphinx's rendering system/RST. The method we are using here -- invisible tables -- might be possible to do natively in our docs, but doing this in HTML is going to be a lot easier.

In GIMP, we can then set guides via Image>Guides>New Guide (by percent) at the following horizontal values:

  • 14.29%
  • 28.57%
  • 42.86%
  • 57.14%
  • 71.43%
  • 85.71%

Then, we can split along those guides using Image>Slice Using Guides

But there's still one problem: Sphinx does not actually support using :doc: links in images. But it does support "external" links via relative paths. For instance, I set the AnVIL logo to link to launch-with/anvil-launch-with.html and that works.
